
在开始之前，重要的事情说三遍!!!!

1. 不要在生产环境使用OpenClaw
2. OpenClaw 有可能会删除数据
3. 可能会绕过模型限制获取隐私

### 准备工作

安装Openclaw 需要nodejs 22+的版本，所以我们要先初始化一些环境，当然最好你的环境里还需要`git`, `curl` 这些软件。



NVM是一个NodeJS的多版本控制器，对于开发人员来说更加需要这个，我们可以在GitHub上边找到这个软件的安装方法。

![image-20260317081039781](https://raw.githubusercontent.com/cloudsmithy/picgo-imh/master/image-20260317081039781.png)



我们可以使用命令行工具来安装NVM：

```
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
```

当然用wget也可以：

```
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
```

然后安装并且切换到最新的NodeJS 的 TLS版本。

```
nvm install 24
nvm use 24
```

然后使用如下命令来进行验证是否安装成功：

```
node -v
npn -v
```

或者你页可以下载NodeJS的离线安装版本，不过从图中看起来他们似乎也推崇nvm的安装办法。

![image-20260317081237353](https://raw.githubusercontent.com/cloudsmithy/picgo-imh/master/image-20260317081237353.png)

### 安装Openclaw

openclaw的官网推荐了源码编译的办法，我们来试一试。文档在此：

https://docs.openclaw.ai/start/getting-started#windows-powershell

![image-20260317081339372](https://raw.githubusercontent.com/cloudsmithy/picgo-imh/master/image-20260317081339372.png)

如果你的windows或者linux，可以使用curl来执行。

```
curl -fsSL https://openclaw.ai/install.sh | bash
```

windows用户可以使用Powershell来进行安装。

```
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 如何卸载

（这个是我在MacOS下的环境，其他的系统需要酌情修改）

openclaw 除了一个一件卸载功能，不过这个和用杀毒软件卸载自己差不多。

```
openclaw uninstall --all --yes --non-interactive
```

如果你看到下面这个文字，大致意思 OpenClaw 已经大体卸载干净了，但全局 CLI 还留在系统里。

🦞 OpenClaw 2026.3.13 (61d171a) — If you're lost, run doctor; if you're brave, run prod; if you're wise, run tests.

Recommended first: openclaw backup create
Gateway service not loaded.
Removed ~/.openclaw
Removed ~/.openclaw/workspace
Removed /Applications/OpenClaw.app
CLI still installed. Remove via npm/pnpm if desired.

你这次卸载结果里，已经删掉了这些东西：

~/.openclaw
~/.openclaw/workspace
/Applications/OpenClaw.app

同时这句：

Gateway service not loaded.

通常表示当时网关服务本来就没在运行，所以没有可停/可卸载的已加载服务；这不一定是报错。官方/社区卸载说明里本来也会先尝试 stop/uninstall gateway，再删除状态目录和 App。

真正还没删的是这部分：

CLI still installed. Remove via npm/pnpm if desired.

也就是你系统 PATH 里的 openclaw 命令大概率还在

用这个检查下进程确认不再运行：
```
 ps aux | grep -i openclaw
```

然后在补上这几个命令：
```
npm uninstall -g openclaw    
rm -rf ~/.openclaw 
rm -rf /tmp/openclaw/
```
