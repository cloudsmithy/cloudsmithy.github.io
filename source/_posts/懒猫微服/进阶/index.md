入门篇帮你把懒猫微服跑起来了，但那只是开始。进阶篇要带你深入系统内核，解锁那些官方文档不会告诉你的高级玩法。这里的每一篇文章，都是我在实际使用中反复验证、踩坑总结出来的干货。

系列开篇是跨架构镜像构建的踩坑记录。如果你用 Apple Silicon 的 Mac 开发应用，想要上架到懒猫商店，一定会遇到 ARM 和 X86 架构不兼容的问题。我详细记录了使用 Docker Buildx 构建多架构镜像的全过程，包括各种报错的解决方案。这篇文章能帮你省下至少一天的调试时间。

接下来是懒猫微服最硬核的内容之一：Docker 多引擎共存机制。你知道懒猫微服上其实跑着三套 Docker 吗？系统 Docker、应用商店 Docker、用户 Docker，它们是如何共存的？如何在不影响系统的情况下自己折腾？这两篇文章会给你答案。

进阶篇还深入讲解了 OIDC 单点登录的接入方式。懒猫微服自带 OIDC Provider，你可以用它实现所有应用的统一登录。我不仅讲了如何接入懒猫自带的 OIDC，还介绍了如何部署 Casdoor 作为独立的身份认证中心，玩法更加灵活。

网络方面，进阶篇教你如何把懒猫微服当作旁路由使用，让局域网内的设备无需安装客户端就能直接访问懒猫上的服务。还有 Docker Context 的使用技巧，让你不用 SSH 登录就能远程管理容器。

如果你对虚拟化感兴趣，WebVirtCloud 安装 Windows 的教程不容错过，包括 virtIO 驱动的各种注意事项。

进阶篇的每一篇文章都需要一定的技术基础，但只要你跟着入门篇走过来，理解起来不会有太大障碍。这些内容会让你对懒猫微服的理解更上一层楼，也能解锁更多高级玩法。

我写这个系列的初衷，是希望把自己踩过的坑都记录下来，让后来者少走弯路。进阶篇的内容可能比入门篇更烧脑，但收获也更大。当你能够自如地在懒猫微服上构建多架构镜像、配置 OIDC 单点登录、玩转 Docker 多引擎，你就已经从普通用户进化成了高级玩家。

这个阶段的学习，不仅仅是掌握几个技巧，更是理解懒猫微服设计哲学的过程。为什么要有三套 Docker？为什么要内置 OIDC？这些设计背后都有深思熟虑的考量。理解了这些，你才能真正把懒猫微服用到极致。

准备好了吗？让我们开始进阶之旅。
