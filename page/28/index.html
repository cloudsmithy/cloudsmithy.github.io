📦 容器篇

写给懒猫微服玩家的 Docker 小书，用故事的方式讲解容器基础。

容器篇是一个特别的系列。它不是传统的技术教程，而是用故事的形式来讲解 Docker 的核心概念。如果你之前对容器一无所知，或者看过一些教程但还是似懂非懂，这个系列会给你全新的理解方式。

第一篇《无法部署的诅咒》从一个常见的问题开始：为什么同样的代码，在我电脑上能跑，到服务器上就不行了？这个问题引出了容器技术诞生的背景和它要解决的核心问题。读完这篇，你会理解为什么我们需要 Docker。

第二篇《镜像旅馆的秘密》讲的是 Docker 镜像。镜像是容器的基础，但它到底是什么？和虚拟机镜像有什么区别？为什么镜像可以分层？这篇文章用”旅馆”的比喻，把这些概念讲得通俗易懂。

第三篇《容器部落生活》深入容器的运行时。容器启动后发生了什么？容器之间如何隔离？容器和宿主机是什么关系？这篇文章会带你走进容器的内部世界。

第四篇《数据之岛与持久化卷》解决一个关键问题：容器是临时的，但数据需要持久保存，怎么办？这篇文章详细介绍了 Docker 的数据卷机制，以及各种数据持久化的最佳实践。

第五篇《多容器交响曲：Docker Compose 上场》是系列的收官之作。现实中的应用往往需要多个容器协作，比如 Web 服务器 + 数据库 + 缓存。Docker Compose 就是用来编排多容器应用的工具，这篇文章会教你如何使用它。

容器篇的五篇文章形成了一个完整的学习路径，从”为什么”到”是什么”再到”怎么用”，循序渐进。读完这个系列，你就具备了在懒猫微服上玩转 Docker 的基础知识。

🛠️ 开发篇

想给懒猫商店贡献应用？开发篇就是你的完整上架指南。

懒猫微服的应用生态离不开社区开发者的贡献。如果你有一个好用的开源项目，或者自己写了一个小工具，都可以打包上架到懒猫商店，让更多人受益。开发篇会手把手教你完成这个过程。

系列从”上架应用需要哪些知识”开始，帮你建立整体认知。懒猫应用的打包并不复杂，核心就是三个文件：lzc-build.yml、lzc-icon.png、lzc-manifest.yml。我会详细解释每个文件的作用和写法，让你快速上手。

第一篇正式教程是全栈应用的上架指南，以一个前后端分离的项目为例，演示从打包脚本编写到 LPK 生成的完整流程。你会了解到懒猫应用的路由规则、静态文件托管、后端服务启动等核心概念。

Devshell 是懒猫提供的远程开发环境，开发篇专门用一篇文章介绍它的使用方法。有了 Devshell，你可以直接在懒猫微服上调试应用，不用在本地搭建复杂的开发环境。

如果你已经有一个跑在 Docker Compose 上的应用，开发篇会教你如何将它移植到懒猫微服。从 Compose 文件到 Manifest 的转换、镜像源的处理、端口和路由的配置，每一步都有详细说明。

OIDC 接入是开发篇的重头戏，分为上下两篇。懒猫微服自带 OIDC Provider，你的应用可以直接接入实现单点登录，用户体验会好很多。我会从 OIDC 的基本概念讲起，一直到代码实现，确保你能完全理解并落地。

开发篇还涵盖了反向代理配置、跨域问题解决、Docker Compose Override 的使用、自签名 SSL 证书的处理等实战内容。这些都是开发过程中经常遇到的问题，提前了解能省去很多排查时间。

🔧 排查篇

踩坑记录与问题排查，希望我踩过的坑你不用再踩。

排查篇收录的是我在使用懒猫微服过程中遇到的各种问题和解决方案。这些问题有的是懒猫微服特有的，有的是 Docker 或 Linux 通用的，但都是在懒猫微服这个环境下遇到并解决的。

第一篇记录的是应用商店 pip 使用清华源报错 403 Forbidden 的问题。这是一个很诡异的问题，同样的清华源地址，在本地能用，在懒猫微服的应用容器里就报 403。经过一番排查，最终找到了原因和解决方案。这篇文章详细记录了整个排查过程，包括如何查看容器日志、如何定位问题、以及最终的解决方法。

第二篇是关于上架应用后 pip 安装报错 HASH 不一致的问题。这个问题发生在应用打包上架之后，用户安装时 pip 报错说包的 HASH 值和预期不符。问题的根源和解决方案都在文章里，如果你也在开发懒猫应用，这篇文章值得一看。

第三篇记录的是一个网络相关的问题：Mac 上 Orbstack 的 Docker 容器访问懒猫微服上的 WordPress 时，出现 IPv6 解析问题。这个问题比较小众，但如果你正好遇到了类似的情况，这篇文章可能会帮到你。

排查篇的文章数量不多，因为不是所有问题都值得写成文章。我收录的都是那些排查过程比较曲折、解决方案不太容易找到的问题。每一篇都包含了完整的问题描述、排查思路和最终方案，希望能帮你节省排查时间。

如果你在使用懒猫微服的过程中遇到了什么奇怪的问题，也欢迎和我交流。说不定你的问题会成为排查篇的下一篇文章。

📖 故事篇

技术之外，还有这些有趣的故事。

故事篇记录的是我使用懒猫微服过程中发生的真实经历。这些文章没有太多技术细节，更多是分享一些使用场景和心得体会，读起来会轻松很多。

第一个故事发生在一次服务器宕机之后。当时公司的开发服务器挂了，我和前端同事急需一个临时的开发环境。情急之下想到了家里的懒猫微服，通过穿透服务把开发环境暴露出来，两个人远程结对编程，竟然顺利完成了当天的工作。这次经历让我意识到，懒猫微服不只是一个家用 NAS，关键时刻还能救急。

第二个故事是在西湖边发生的。一次偶然的机会，认识了一个对 NAS 感兴趣的朋友。她之前完全没接触过这类设备，我就用懒猫微服作为例子，手把手教她什么是 NAS、能做什么、怎么用。从最基础的概念讲起，到实际操作演示，整个过程让我重新审视了懒猫微服的易用性。

第三个故事是帮朋友做内网穿透。朋友有一台没有公网 IP 的服务器，想从外网访问上面的服务。传统方案要么买云服务器做跳板，要么用 frp 之类的工具自己搭建。我直接用懒猫微服的转发功能帮他解决了问题，整个过程不到十分钟。

第四个故事是我自己从折腾内网穿透到使用懒猫微服的心路历程。之前为了从外网访问家里的服务，我尝试过各种方案：花生壳、frp、Tailscale…每一个都有各自的问题。直到用上懒猫微服，才发现原来内网穿透可以这么简单。

故事篇的更新频率不高，因为好故事需要时间积累。但每一篇都是真实发生的，希望能给你一些启发和共鸣。

✨ 炫技篇

懒猫微服还能这么玩？炫技篇收录了一些脑洞大开的骚操作。

这个系列的文章不是必需品，但绝对是调味剂。如果你已经把懒猫微服用得很顺手了，想看看还有什么新鲜玩法，炫技篇会给你惊喜。

第一篇是车机远程听歌的方案。如果你有黑群晖，又不想洗白，可以用懒猫微服的穿透服务把 DS Music 暴露到公网，这样在车上就能用 CarPlay 播放家里 NAS 上的音乐了。整个方案不需要公网 IP，不需要复杂的内网穿透配置，懒猫微服帮你搞定一切。

第二篇是一个有趣的小技巧：用懒猫微服倒推停电时间。懒猫微服会记录系统的启动时间，结合一些简单的计算，你可以知道家里上次停电是什么时候、停了多久。虽然不是什么刚需功能，但关键时刻说不定能派上用场。

第三篇是双人串流打游戏的方案。用 Sunshine 和 Moonlight 这对组合，你可以把懒猫微服变成一个游戏串流服务器。两个人可以同时连接，一起玩本地多人游戏，延迟低到几乎感觉不到。

第四篇是蓝牙音箱的玩法。懒猫微服可以连接蓝牙音箱，变成一个音乐播放终端。配合一些自动化脚本，你可以实现语音控制、定时播放等功能。文章详细介绍了蓝牙配对、音频输出配置、开机自启等技术细节。

炫技篇的文章数量不多，但每一篇都是我自己折腾出来的真实案例。这些玩法可能不适合所有人，但如果正好戳中你的需求，那就是意外之喜。未来我还会继续更新更多有趣的玩法，敬请期待。

🖥️ 算力仓

懒猫算力仓系列教程，带你玩转这台 AI 算力设备。

算力仓是懒猫推出的一款专注于 AI 计算的设备，可以和懒猫微服配合使用，也可以独立运行。这个系列记录了我从开箱到深度使用的全过程。

第一篇是开箱手记，详细介绍了算力仓的硬件配置、外观设计和初次开机体验。作为一款 AI 算力设备，算力仓的配置还是很有诚意的，文章里有详细的参数和实拍图。

第二篇讲的是如何将算力仓绑定到懒猫微服。这是使用算力仓的第一步，绑定之后你就可以通过懒猫微服的界面来管理算力仓，包括查看状态、部署应用等。整个绑定过程需要用到懒猫官方的 AI 浏览器，文章详细介绍了每一个步骤。

第三篇是 SSH 配置教程。和懒猫微服一样，算力仓也支持 SSH 远程登录。但算力仓的系统和懒猫微服有所不同，配置方式也略有差异。这篇文章会教你如何开启 SSH、配置免密登录、以及设置免密 sudo，让你能够更方便地管理算力仓。

第四篇讲的是文件传输。算力仓主要用于 AI 计算，经常需要传输大量的模型文件和数据集。这篇文章介绍了几种高效的文件传输方式，包括 SCP、rsync、以及通过懒猫网盘中转等方案，帮你找到最适合自己场景的方法。

第五篇是图形界面的配置。虽然算力仓主要通过命令行操作，但有时候图形界面还是更方便，比如设置静态 IP。这篇文章教你如何给算力仓安装图形界面，以及如何通过远程桌面连接。

算力仓系列还在持续更新中，后续会有更多关于 AI 模型部署、性能调优等内容，敬请期待。

📈 进阶篇

入门篇帮你把懒猫微服跑起来了，但那只是开始。进阶篇要带你深入系统内核，解锁那些官方文档不会告诉你的高级玩法。这里的每一篇文章，都是我在实际使用中反复验证、踩坑总结出来的干货。

系列开篇是跨架构镜像构建的踩坑记录。如果你用 Apple Silicon 的 Mac 开发应用，想要上架到懒猫商店，一定会遇到 ARM 和 X86 架构不兼容的问题。我详细记录了使用 Docker Buildx 构建多架构镜像的全过程，包括各种报错的解决方案。这篇文章能帮你省下至少一天的调试时间。

接下来是懒猫微服最硬核的内容之一：Docker 多引擎共存机制。你知道懒猫微服上其实跑着三套 Docker 吗？系统 Docker、应用商店 Docker、用户 Docker，它们是如何共存的？如何在不影响系统的情况下自己折腾？这两篇文章会给你答案。

进阶篇还深入讲解了 OIDC 单点登录的接入方式。懒猫微服自带 OIDC Provider，你可以用它实现所有应用的统一登录。我不仅讲了如何接入懒猫自带的 OIDC，还介绍了如何部署 Casdoor 作为独立的身份认证中心，玩法更加灵活。

网络方面，进阶篇教你如何把懒猫微服当作旁路由使用，让局域网内的设备无需安装客户端就能直接访问懒猫上的服务。还有 Docker Context 的使用技巧，让你不用 SSH 登录就能远程管理容器。

如果你对虚拟化感兴趣，WebVirtCloud 安装 Windows 的教程不容错过，包括 virtIO 驱动的各种注意事项。

进阶篇的每一篇文章都需要一定的技术基础，但只要你跟着入门篇走过来，理解起来不会有太大障碍。这些内容会让你对懒猫微服的理解更上一层楼，也能解锁更多高级玩法。

我写这个系列的初衷，是希望把自己踩过的坑都记录下来，让后来者少走弯路。进阶篇的内容可能比入门篇更烧脑，但收获也更大。当你能够自如地在懒猫微服上构建多架构镜像、配置 OIDC 单点登录、玩转 Docker 多引擎，你就已经从普通用户进化成了高级玩家。

这个阶段的学习，不仅仅是掌握几个技巧，更是理解懒猫微服设计哲学的过程。为什么要有三套 Docker？为什么要内置 OIDC？这些设计背后都有深思熟虑的考量。理解了这些，你才能真正把懒猫微服用到极致。

准备好了吗？让我们开始进阶之旅。
