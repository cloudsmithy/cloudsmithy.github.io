# GitHub Pages 域名问题修复记录

## 问题

部署后 GitHub Pages 域名自动变成实效的域名，而不是预期的 `*.github.io`

## 原因

1. `source/CNAME` 文件中配置了旧域名 `airag.click`

## 解决方案

### 1. 删除 CNAME 文件

```bash
rm source/CNAME
```

### 3. 重新部署

```bash
hexo clean && hexo generate && hexo deploy
```

## 说明

- CNAME 文件会被 Hexo 复制到生成目录，告诉 GitHub Pages 使用自定义域名
- 删除后将使用默认的 `<username>.github.io` 域名
